# 🤝 Guía de Contribución

¡Gracias por tu interés en contribuir a este proyecto! 🚀  
Antes de enviar un Pull Request (PR), por favor revisa las siguientes pautas.

---

## 📌 Reglas Generales

- Si tienes sugerencias (bugs, mejoras de UI/UX o del README), abre primero un **Issue**.
- Asegúrate de seguir buenas prácticas de desarrollo.
- Cada PR debe estar relacionado con un Issue (`Fixes #número`).
- Revisa ortografía y gramática antes de enviar.
- Crea un PR por cada Issue (no mezclar cambios).

---

## ⚙️ Instalación del Proyecto

1. Clona el repositorio:
   ```bash
   git clone <repo-url>
   cd <nombre-del-proyecto>
 ```
Instala dependencias:
 ```
npm install
 ```
# o
 ```
yarn install
 ```
Configura variables o archivos necesarios (si aplica).

---

## 💻 Guías de Código

- Escribe código limpio y legible.
- Usa nombres de variables claros y descriptivos.
- Agrega comentarios cuando sea necesario.
- Evita romper funcionalidades existentes (compatibilidad hacia atrás).

---

## 🧪 Pruebas

Ejecuta los tests:
 ```
npm test
 ```
# o
 ```
yarn test
 ```

- Asegúrate de que todo funcione correctamente.
- Agrega pruebas para nuevas funcionalidades o fixes.
- Realiza pruebas manuales si hay cambios visuales.

---

## 🔀 Guía para Pull Requests

Crea una nueva rama:
 ```
git checkout -b feature/#numero-issue
 ```
Realiza tus cambios y haz commit:
 ```
git commit -m "feat: descripción del cambio"
 ```
Sube los cambios:
 ```
git push origin feature/#numero-issue
 ```
Crea el Pull Request en GitHub:
 ```
Usa: Fixes #numero_issue
 ```

- Describe claramente los cambios
- Agrega capturas o GIFs si aplica (UI)
- Actualiza la documentación si es necesario.

---
## ✅ Checklist antes de enviar

 - Código revisado y limpio
 - ests pasan correctamente
 - No se rompen funcionalidades existentes
 - Documentación actualizada
 - PR relacionado con un Issue

---

## 🖥️ Alternativa: GitHub Desktop

- Abre GitHub Desktop
- Clona el repositorio
- Cambia a la rama correspondiente
- Realiza cambios en tu editor
- Haz commit con descripción clara
- Haz push al repositorio
- Crea el Pull Request desde GitHub
- Espera revisión del equipo
